# Плагин Z-Mod для устройства IFS Jacker

https://github.com/ninjamida/ifs-jacker
https://www.printables.com/model/1644745-ifs-jacker

Чтобы включить репозиторий внешних плагинов, не разрабатываемых автором Z-Mod, выполните команду

```
ENABLE_EXTRA_PLUGINS
```

Каждый раз после обновления этого плагина следует заново запускать для него макрос `ENABLE_PLUGIN`. Это необходимо по двум причинам:
1. Могут появиться новые скрипты для установки (данный плагин устанавливает Python-скрипты в дополнение к обычным CFG-файлам), которые должен обработать установочный скрипт;
2. После обновлений, как правило, требуется перезагрузка, которую также инициирует установочный скрипт.

```
ENABLE_PLUGIN NAME=ifs_jacker
```

Если у вас установлен этот плагин, но подключён обычный IFS (а не IFS Jacker), ваш IFS будет несколько раз отключаться при запуске Z-Mod. 
После третьего раза это должно прекратиться.
Это происходит из-за того, что плагин отправляет команду, предназначенную для обнаружения IFS Jacker, на которую обычный IFS не может ответить, и поэтому Z-Mod интерпретирует это как отключение.
После трёх неудачных попыток плагин прекращает выполнять эту проверку до тех пор, пока IFS не будет отключён и повторно подключён или принтер не будет перезапущен.

Если вы планируете долгосрочно вернуться к использованию обычного IFS, лучше отключить плагин IFS Jacker.
